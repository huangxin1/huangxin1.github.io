保存到什么地方

和c语言一样，变量保存地方的不一样也决定了程序变量类型。
（1）寄存器。
处理器内部。
需要由编译分配，我们程序对此没有直接的控制权，也不能在自己的程序中找到寄存器存在的任何踪迹。
（2）堆栈。
常规RAM（随机访问存储器）区域，可通过它的“堆栈访问指针”获得处理的直接支持堆栈，堆栈指针若向下移，会创建新的内存；若向上移，则会释放那些内存，这是一种特别快，特别有效的数据保存方式，仅次于寄存器。
创建程序时，Java编译器必须精确地知道堆栈内保存的所有数据的“长度”以及“存在时间”，这是由于它必须生成相应的代码，以便向上和向下移动指针。这一限制无疑影响了程序的灵活性，所以尽管有些Java数据要保存在堆栈里-特别是Java对象句柄，但是Java对象并不放在其中。
（3）堆。
一种唱过用途的内存池（也在RAM区域），其中保存了Java对象，和堆栈不同，“内存堆”和“堆”（Heap）最吸引人的对方在于编译器不必知道要从堆里分配多少存储空间，也不必知道存储的数据要在堆里停留多长时间。因此用堆保存数据会得到更大的灵活性。
Java中，要求创建一个对象时，只需用new命令编制相关的代码即可，执行这些代码的，会在堆里自动进行数据的保存。当然在堆里分配储存空间会花掉更长的时间。JAVA的垃圾回收机制主要处理的就是堆里面的对象。
（4）静态存储。
这里的“静态”（static）是指“位于固定位置”（尽管也在RAM里）。
程序运行期间，静态存储的数据将随时等候调用。可用static关键词指出一个对象的特定元素是静态的。但java对象本身永远都不会置入静态存储空间。
（5）常数存储。
常数值通常直接置于程序代码内部。
这样做是安全的，因为他们永远不会改变。有的常数需要严格的保护，所以可以考虑将它们置入ROM（只读存储器）。
（6）非RAM存储。
若数据完全独立于一个程序之外，则程序不运行时仍然可以存在，并在程序的控制范围之外。
Java中两个最主要的例子便是“流式对象”和“固定对象”。对于流式对象，对象会变成字节流，通常会发给另一台机器。而对于固定对象，对象保存在磁盘里。即使程序终止运行，他们仍然可保持自己的状态不变。对于这些类型的数据存储，一个特别有用的技巧就是他们能存在其他媒体中。一旦需要，甚至可以将他们恢复成普通的、基于RAM的对象。

特殊情况：主要类型

就像c语言里面有的基本数据类型一样。
对象创建置于“堆”中，主要类型变量是一个并非“句柄”的自动变量，放置于堆栈中，可以高效的存取。
(寄存器中) > 句柄、主要类型（堆栈中）> 对象（堆中）> 非RAM存储（流式对象、固定对象）

主数据类型也拥有自己的“封装器”（wrapper）类。这意味着假如想让堆内的对象表示主要类型，就要使用对应的封装器。
